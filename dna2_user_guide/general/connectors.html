Connectors

Connectors, as the name suggests, are file that help you connect with data sources.
Each helper file is simply an object with methods for a particular data source.

DNA² does not load Helper Files by default, so the first step in using a Helper is to load it. Once loaded, it becomes globally available in your controller and views.

Loading a Connector

Connectors are automatically loaded if a model needs them.

Writing a Connector

Connector files are stored in your system/application/models/bpm/connectors, and the rule to name them is $name_connector.php.
i.e.: for a generic mysql you will have the file mysql_connector.php

You cant take the bundled connector as an example for making new ones, as long as we add connectors, they will be available in future releases.

Using a Connector

In order to use a connector you have to define a Data Object in your model, this object will make the data available to the process.
DataObject properties:
Connector: the name of the connector i.e: mysql
Query: a query your connector can execute.

*** mongo only ** datastoreref: The collection wich holds the data.

Note: datastoreref and itemsubjectref will be passed to connectors so you can write your connector and give them some use.

Loading Multiple connectors

If you need to load more than one connector the only thing you have to do is to create a data object for each one in your model
